## 2. Tacotron2 TTS Test

코랩에서 사용했던 Tacotron2 모델을 로컬 환경에서도 테스트해봤다.

여러 문장을 입력해 음성 합성 결과를 확인했고, 일부 문장은 비교적 잘 나왔지만 반복 발화나 특정 단어에서는 발음이 무너지거나 어색해지는 현상이 있었다.  
특히 반복 문장에서 attention collapse처럼 보이는 출력이 나와서, TTS 모델이 항상 안정적으로 발화하는 것은 아니라는 점을 체감했다.

### Result
- 로컬 TTS 실행 성공
- 일부 문장 출력 정상
- 반복 문장/특정 발음에서 품질 저하 확인

### Notes
- 발음이 의도와 다르게 출력되는 문제가 있었음
- 모델 출력 품질은 입력 문장 형태에 영향을 많이 받음

## 4. Desktop Assistant Pipeline

학원에서는 실제로 계속 말하면서 테스트하기가 부담스러워서, 텍스트 명령 기반 방식도 함께 구현했다.

단순 문자열 비교가 아니라, 다양한 한국어 표현을 하나의 동작으로 연결할 수 있도록 `parse_command` 구조를 만들었다.

예를 들어:
- "소리좀키워봐"
- "음량올려"
- "볼륨업"

같은 표현을 모두 같은 명령으로 처리할 수 있도록 설계했다.

이후 파싱된 명령을 실제 시스템 동작으로 연결했다.

### Implemented Features
- 메모장 실행
- 볼륨 조절
- 스크린샷 저장
- 미디어 제어 구조 설계

### Pipeline
텍스트 입력 → 명령 해석 → 시스템 동작 실행 → 음성 결과 출력

### What I Learned
단순히 STT만 붙인다고 음성 어시스턴트가 되는 게 아니라,  
**입력 → 해석 → 실행 → 피드백**까지 이어져야 실제 기능이 된다는 점이 중요했다.
