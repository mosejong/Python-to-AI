## 1. 정규표현식 기초

정규표현식은 문자열 패턴을 찾기 위한 규칙이다.

### 주요 기호
- `^` : 문자열의 시작
- `$` : 문자열의 끝
- `.` : 임의의 한 문자
- `\s` : 공백 문자
- `\S` : 공백이 아닌 문자
- `\d` : 숫자
- `\D` : 숫자가 아닌 문자
- `\w` : 알파벳, 숫자, 언더스코어(`_`)
- `\W` : `\w`가 아닌 문자
- `\n` : 개행 문자
- `\t` : 탭 문자

### 수량 한정자
- `?` : 0개 또는 1개
- `*` : 0개 이상
- `+` : 1개 이상
- `{n}` : n개
- `{n,m}` : n개 이상 m개 이하
- `{n,}` : n개 이상

### 그룹 패턴
- `()` : 여러 패턴을 하나의 그룹으로 묶을 때 사용

## 2. grep 명령어

`grep`은 파일에서 특정 문자열이나 패턴이 포함된 행을 검색하는 명령어다.

### 기본 형식

```bash
grep 옵션 패턴 파일명
```

### 주요 옵션
- `-i` : 대소문자 구분 없이 검색
- `-v` : 패턴과 일치하지 않는 행 출력
- `-c` : 패턴과 일치하는 행 개수 출력
- `-w` : 단어 단위로 정확히 매칭
- `-n` : 라인 번호 함께 출력

### 실습 예시

```bash
grep "hello" sample.txt
grep -n "error" log.txt
grep -i "linux" note.txt
grep -v "fail" result.txt
```

### 정리
`grep`은 단순 문자열 검색뿐 아니라 정규표현식과 함께 사용하면 훨씬 강력하다.

## 3. File Redirection

리다이렉션은 표준 입력, 출력, 오류의 흐름을 파일로 재지정하는 기능이다.

### 표준 스트림
- `stdin` : 표준 입력
- `stdout` : 표준 출력
- `stderr` : 표준 에러 출력

### 파일 디스크립터
- `0` : 표준 입력
- `1` : 표준 출력
- `2` : 표준 에러

### 리다이렉션 연산자
- `>` : 표준 출력을 파일로 저장
- `>>` : 기존 내용 뒤에 이어서 저장

### 실습 예시

```bash
echo hello > a.txt
echo world >> a.txt
ls not_exist 2> error.txt
```

### 정리
출력과 에러를 분리해 저장할 수 있어서 로그 관리나 디버깅에 유용하다.

---

## 4. Pipe

파이프(`|`)는 한 명령어의 출력을 다른 명령어의 입력으로 넘기는 기능이다.

### 실습 예시

```bash
ps -ef | grep python
cat sample.txt | grep hello
ls | grep ".txt"
```

### 정리
여러 명령어를 연결해 한 번에 처리할 수 있어서 리눅스 작업 효율을 크게 높여준다.

---

## 5. mount

마운트는 저장장치를 특정 디렉토리에 연결하는 작업이다.

윈도우는 보통 자동 연결되지만, 리눅스는 직접 마운트가 필요한 경우가 있다.

### 기본 형식

```bash
mount 옵션 디바이스 디렉토리
```

### 주요 옵션
- `-a` : `/etc/fstab`에 명시된 파일 시스템 전체 마운트
- `-t` : 파일 시스템 유형 지정
- `-o` : 추가 옵션 설정

### 관련 명령어

```bash
remount 디바이스 디렉토리
df
```

## 6. 프로세스

프로세스는 메모리에 적재되어 실행 중인 프로그램이다.

### 특징
- 프로그램 실행 시 하나 이상의 프로세스를 가짐
- 여러 프로세스가 병행 실행 가능
- 부모/자식 프로세스 관계가 존재
- 커널에 의해 관리됨
- 각 프로세스는 소유자와 PID를 가짐

### 참고
- PID 1 : `init`
- PID 2 : `kthreadd`

---

## 7. 메모리 구성

프로그램 실행 시 메모리는 여러 영역으로 나뉜다.

- **코드 영역** : 프로그램 코드 저장
- **데이터 영역** : 전역 변수, 정적 변수
- **BSS 영역** : 초기화되지 않은 변수
- **힙(Heap)** : 동적 메모리 할당
- **스택(Stack)** : 지역 변수, 매개변수, 복귀 주소
- **커널 영역** : 운영체제가 사용하는 영역

---

## 8. 프로세스 관련 명령어

### ps
현재 실행 중인 프로세스를 확인하는 명령어

### 주요 옵션
- `-e` : 전체 프로세스 출력
- `-f` : 상세 정보 출력
- `-a` : 전체 사용자 프로세스 출력
- `-u` : 사용자와 시작 시간 정보 출력
- `-x` : 터미널 제어 없는 프로세스 확인

### 실습 예시

```bash
ps
ps -ef
ps aux
```

### kill
프로세스를 종료하는 명령어

### 주요 옵션
- `-l` : 시그널 목록 출력
- `-1` : 재실행
- `-9` : 강제 종료
- `-15` : 정상 종료

### 실습 예시

```bash
kill -9 1234
kill -15 1234
```

## 9. job

`job`은 현재 터미널에서 백그라운드로 실행 중인 작업을 보여준다.

### 특징
- 현재 터미널 세션 기준으로 관리
- 터미널 종료 시 함께 종료됨
- 시스템 전체 프로세스와는 개념이 다름

---

## 10. at

`at`은 지정된 시간에 1회 실행되는 작업 예약 명령어다.

### 기본 형식

```bash
at 옵션 시간 날짜 +증가시간
```

### 주요 옵션
- `-m` : 작업 완료 시 메일 전송
- `-f` : 스크립트 파일 실행
- `-l` : 예약 작업 목록 출력
- `-v` : 실행 시간 출력
- `-d` : 예약 작업 삭제

---

## 11. crontab

`crontab`은 주기적으로 반복 실행되는 작업을 등록할 때 사용한다.

### 주요 옵션
- `-l` : 현재 crontab 보기
- `-e` : 현재 crontab 수정
- `-r` : 현재 crontab 전체 삭제
- `-u` : 특정 사용자 crontab 관리

### 작성 형식

```bash
분 시 일 월 요일 명령어
```

예시:

```bash
0 9 * * * python backup.py
```

---

## 12. SSH

SSH는 원격 컴퓨터에 안전하게 접속하거나 명령을 실행할 수 있도록 해주는 프로토콜이다.

### 관련 개념
- `Telnet` : 기능은 비슷하지만 보안이 취약함
- `OpenSSH` : 우분투에서 SSH 사용 시 대표 패키지

## 💻 실습하면서 느낀 점

- `grep`과 정규표현식은 단순 검색을 넘어서 로그 분석이나 파일 탐색에 매우 유용할 것 같았다.
- 리다이렉션과 파이프는 명령어를 조합해서 더 강력하게 쓸 수 있다는 점이 인상적이었다.
- `mount`, `at`, `crontab`은 아직 낯설지만 서버나 Docker를 배울 때 계속 연결될 개념이라는 생각이 들었다.
- 프로세스와 메모리 구조를 함께 배우면서 운영체제가 프로그램을 관리하는 흐름을 조금 더 구조적으로 이해할 수 있었다.

---

## 어려웠던 점

- 정규표현식 기호가 비슷해서 처음에 헷갈렸다.
- `ps`, `kill`, `job` 명령어는 각각 쓰는 상황을 구분하는 연습이 더 필요하다.
- 시스템 관리 명령어는 개념만 보면 이해되지만, 손에 익으려면 직접 여러 번 쳐봐야 할 것 같다.
