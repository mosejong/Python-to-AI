Input Embedding

입력 문장의 각 단어를 벡터로 바꾸는 시작 단계입니다. 사람이 쓰는 단어를 모델이 처리할 수 있는 수치 표현으로 바꿉니다.

입력: 토큰 인덱스
출력: 임베딩 벡터
느낌: 단어를 좌표로 바꾸는 단계
현재 단계에서 토큰이 어떻게 보이는지
예시 입력: [나는, 밥을, 먹었다]
현재 표현: 각 단어를 임베딩 벡터로 변환하는 단계
Q · K · V 시각화

Attention에서 가장 많이 헷갈리는 Q, K, V를 역할 중심으로 바로 이해할 수 있게 분리했습니다.

Q · Query

지금 내가 무엇을 찾고 싶은지를 나타냅니다. 디코더 기준으로 보면 “지금 만들 단어와 관련된 정보가 뭐지?”라는 질문에 가까워요.

K · Key

각 단어가 가진 단서입니다. Query가 이 Key들과 비교되면서 어떤 단어를 얼마나 참고할지 점수가 계산됩니다.

V · Value

실제로 가져올 내용입니다. Attention score가 높게 나온 단어일수록 해당 Value 정보가 더 많이 반영됩니다.

쉽게 말하면
Q = 지금 찾는 질문 / K = 비교할 표지판 / V = 실제 가져올 내용
번역 예시
디코더가 I를 만들 때 Q는 “주어 정보 뭐지?”를 찾고, 인코더 쪽 Key 중 나는와 잘 맞는 곳의 Value를 더 많이 가져옵니다.
번역 예시로 보면 더 쉬움

나는 밥을 먹었다 → I ate rice 기준으로 흐름을 단순화해서 정리했습니다.

입력
문장을 토큰화해서 [나는, 밥을, 먹었다]로 나눕니다.
인코더
각 단어가 서로 어떤 관계인지 계산해서 문맥이 담긴 표현으로 바꿉니다.
디코더 시작
[START] 토큰부터 시작해서 첫 단어를 예측합니다.
Cross-Attention
예를 들어 I를 만들 때 입력의 나는 쪽 정보에 집중합니다.
출력 생성
I → ate → rice → [END] 순으로 한 단어씩 생성합니다.
딱 헷갈리는 포인트만

왜 RNN보다 빠를까

RNN은 순서대로 읽어야 하지만 Transformer는 단어들을 동시에 보고 계산할 수 있어서 병렬 처리에 훨씬 유리합니다.

왜 Positional Encoding이 필요할까

Transformer는 순차적으로 읽지 않기 때문에, 단어 위치를 따로 알려주지 않으면 순서를 구분할 수 없습니다.

Cross-Attention이 왜 중요할까

디코더가 현재 만들고 있는 출력 단어와 관련된 입력 부분을 정확히 참고하게 해주는 핵심 연결 고리입니다.
