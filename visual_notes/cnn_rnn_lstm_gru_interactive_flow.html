LSTM / GRU 왜 나왔나

RNN은 긴 시퀀스에서 앞 정보가 약해질 수 있어서, 기억을 더 잘 관리하는 구조인 LSTM과 GRU가 등장했습니다. 이번에는 단순 글이 아니라 게이트가 실제로 무엇을 조절하는지가 보이도록 구성했습니다.

LSTMforget, input, output gate와 cell state를 따로 둬서 기억을 세밀하게 관리합니다.
GRUupdate gate와 reset gate로 LSTM보다 단순하게 기억을 조절합니다.
핵심 차이LSTM은 cell state와 hidden state를 나눠 관리하고, GRU는 hidden state 중심으로 더 단순하게 처리합니다.
복습 포인트RNN의 기억 약점을 보완한 게 LSTM, 그걸 더 가볍게 만든 게 GRU입니다.
LSTM 시각화

LSTM은 cell state와 hidden state를 분리해서 기억을 관리합니다. 아래 흐름만 정확히 이해해도 절반은 끝입니다.

Forget Gate
이전 cell state에서 어떤 기억을 버릴지 결정
Input Gate
현재 입력에서 어떤 새 정보를 저장할지 결정
Cell State
장기 기억 통로. 과거 기억과 새 정보를 합쳐 계속 전달
Output Gate
현재 시점에 무엇을 밖으로 보여줄지 결정
LSTM 기억 흐름
INPUTS
x_t + h_(t-1) + C_(t-1)
현재 입력, 이전 hidden state, 이전 cell state가 함께 들어옵니다.
FORGET GATE
버릴 기억 선택
이전 기억 중 지금 필요 없는 부분은 약하게 만듭니다.
INPUT GATE
새로 저장할 기억 선택
현재 입력에서 중요한 정보만 골라 cell state에 추가합니다.
CELL STATE
장기 기억 유지
버릴 것과 새로 넣을 것을 반영한 장기 기억 통로입니다.
OUTPUT GATE
현재 출력 결정
업데이트된 기억 중에서 지금 밖으로 보여줄 hidden state를 만듭니다.
GRU 시각화

GRU는 LSTM보다 단순합니다. cell state를 따로 두지 않고, hidden state 하나 중심으로 기억을 조절합니다.

Update Gate
이전 기억을 얼마나 유지할지, 새 정보를 얼마나 섞을지 결정
Reset Gate
이전 기억을 얼마나 무시하고 새 입력에 집중할지 결정
Hidden State
LSTM보다 단순한 구조로 기억을 갱신하며 다음 시점으로 전달
GRU 기억 흐름
INPUTS
x_t + h_(t-1)
현재 입력과 이전 hidden state가 들어옵니다.
RESET GATE
이전 기억 얼마나 무시할까
필요 없는 과거 정보는 덜 보게 해서 새 입력 중심으로 계산할 수 있게 합니다.
UPDATE GATE
이전 기억 얼마나 유지할까
기존 기억을 유지할지, 새 입력을 많이 반영할지 결정합니다.
NEW HIDDEN STATE
단일 hidden state 갱신
LSTM보다 단순하게 다음 시점으로 보낼 기억을 완성합니다.
RNN vs LSTM vs GRU 한눈에
RNN구조 단순. 긴 기억 약함
LSTM기억 관리 정교. cell state와 gate가 있어 구조가 복잡
GRULSTM보다 단순. gate 수가 적고 계산이 비교적 가벼움
복습 포인트RNN의 기억 약점을 보완한 게 LSTM, 이를 더 단순화한 게 GRU
